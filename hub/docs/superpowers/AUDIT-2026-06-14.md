## Open / follow-up items

1. **Dead code (cosmetic):** the four old tab *bodies* (`#tab-bench`, `#tab-score`,
   `#tab-shootout`, `#tab-gauntlet`) and their JS remain in `index.html`, hidden and
   unreachable (no nav entry). The JS parses and is internally consistent, so it is harmless,
   but it is ~600 lines of cruft. **Recommended:** remove in a separate focused pass (delete
   the bodies + their loader/handler/`addEventListener` JS together, then re-run the suite and
   load the page). Deferred from Task 7 to avoid breaking the load-time wiring mid-rework.
2. **Compare/shootout mapping is best-effort:** the shootout result file shape was not fully
   read; `compareResult()` defensively tries `score`/`coverage`/`summary.pass_rate`. Verify
   against a real shootout run and tighten the mapping if the numbers look off.
3. **Gauntlet "fast" depth** currently runs the whole set (no subset) — the run endpoint
   supports `only:[ids]` but the sets list exposes only counts, not ids. Wire a real subset if
   fast gauntlet runs matter.
