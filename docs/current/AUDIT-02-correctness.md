# AUDIT-02 — Correctness, Determinism & Robustness

**Scope:** `foundry/`, `foundry/blender/`, `foundry/godot_template/` (`.gd` + `.tscn`), `hub/`, `foundry/tests/`.
**Method:** static reads of the spine + cache layers; no execution, no code changes.
**Calibration pillars:** DETERMINISM IS SACRED (cache keys are authoritative when bakes aren't bit-exact); OFFLINE/local (no web-security; do report resource-exhaustion).

The Round-1 audit (`AUDIT-01-architecture.md`) flagged broad structural issues A1–A20. This round follows the structural decomposition it recommended — every finding below cites a file:line, a measurable count where possible, and a concrete one-sentence fix terse enough to ship as a CLI prompt.

## 4. Round summary

The single highest-risk correctness/determinism issue is **C4 (dialogue validator passes when the category needed by the winnability check is missing)**: a quest spec that passes `objective_winnable` *and* `dialogue validation` is silently unwinnable-by-text — the player is told to fetch X while the manifest's target is Y.
