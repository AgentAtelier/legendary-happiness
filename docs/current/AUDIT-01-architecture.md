**Scope:** `foundry/`, `foundry/blender/`, `foundry/godot_template/` (`.gd` + `.tscn`), `hub/`, `foundry/tests/`.
**Inventory:** 57,565 total Python LOC across `foundry/` + `hub/` (excluding `.venv`). Test suite is ~50% of that volume.

## 3. Round summary

The single highest-leverage structural change is **A1+A2+A5**: decompose `scene_compiler.py` (2303 LOC) into ~6 focused modules and extract a shared `tscn_writer.py` + `lighting_bake.py` so that `scene_compiler.py` (slimmer) and `exterior_compiler.py` (slimmer) stop independently re-implementing .tscn emission, bake contracts, and outdoor-room composition ‚Äî eliminating a class of subtle "Godot gates red while Python tests green" bugs that have already cost the engine two prior bundles.

**Other observations**
- **Spine integrity is mostly intact**: `prompt ‚Üí Interpreter ‚Üí Brief ‚Üí planners ‚Üí compile_scene ‚Üí scaffold ‚Üí Godot + build_report` is end-to-end traceable. The muddiest layers are `_LIGHT_EMITTING` (A3) and the double-`ensure_room_shell` / double-`_pre_import` in scaffold (A7).
- **Circular-ish imports**: A9 documented; Python's cache masks it today. Splitting `themes.py` out of `room_control.py` will harden it.
- **Python-build / Godot-runtime data contract** (A11, A12, A13) is the highest-risk soft-area independent of these structural findings; flagged here so Round 2 (determinism) and Round 3 (correctness) can target it.
- **No true orphans** in the Python build path. `world_events.py` and `npc_sim.py` are loose (build-time-only generational functions), but they are invoked (A13).

**Audit disclaimer.** LOC counts are from `wc -l` over the scope dirs; line numbers are from `grep -n` over the structural symbols cited. All findings have file:line evidence; no execution-level introspection was performed.
