# AUDIT-05 — Targeted Performance

Severity: Critical = correctness/wrong-confidence; High = real build-time or runtime cost, real OOM risk on adversarial input; Medium = measurable but bounded cost; Low = nits.
Scope: foundry/, hub/, scripts/ — same as Round 1, with explicit attention to `compile_scene` emitted .tscn (runtime cost) and `subprocess` / Blender process graph (build-time cost).
Cost axes labeled per finding:
- **BUILD** = wall-clock that hits the developer's iteration loop (Blender spawns, Cycles bakes, Godot imports)
- **RUNTIME** = in-Godot cost (FPS, memory, draw calls, node count)

## Round summary

The single highest-leverage build-time change is to **collapse scaffold.py's two `godot --headless --import --quit` invocations into one combined-after-all-copy pass (P1)**, eliminating ~10-15 s of redundant imports per build and unlocking a `PYTHONHASHSEED`-stable scene-compile profile; the second-most-leverage is fixing `lighting_bake._placements_sig` (P2) so GLB content (not filename) drives the cache key, preventing stale-vertex-color serves across re-bakes.

— 27 findings: 2 Critical (P1 double-import + P2 cache-key bubble), 11 High (P3 Cycles bake cost, P4 triplanar on small props, P5 18-shadow-map interior grid, P6 200+ scatter nodes, P7 unbounded Blender pool, P8 O(N²) AABB separation, P9 dead NPC rig nodes per NPC, P10 HIP/CPU silent fallback, P11 lighting-defaults duplication × 7 vars × 2 sites, P12 room_shell called twice), 11 Medium, 3 Low. Build-time axes dominate the impact list (P1, P2, P3, P7, P10, P12). Runtime axes (P4, P5, P6, P9) are 4–5 FPS-class wins concentrated in the npc.gd + interior-lights + triplanar+small-prop intersection.
